# Kit v3.4 — Gemini 退役 + Claude-native Research + v3.3 收尾（design）

- 日期：2026-07-04
- 狀態：使用者已核可（brainstorming 全程分段確認），並授權一路執行到完成
- 前置：v3.3 harness 層已合入 main（7fa1418）；本輪為「收尾 + 結構簡化」

## 1. 背景與動機

1. **Gemini 退役（本輪主軸）**：使用者決定將 Gemini CLI 整合自 kit 移除。
   明確記錄：**這是個人因素的決策，不是模型能力問題**。研究（research）
   能力保留，改由 Claude 原生完成。
2. **v3.3 fix-later 三小項**一直未排程：`--help` 噪音、settings.json
   deploy-if-absent、smoke env 過濾。與 gemini 移除動同一批檔案
   （init.sh、smoke.sh），合併處理。
3. **Repo 衛生**：v3.3 遺留的 stale worktree/branch、Windows Zone.Identifier
   殘檔、untracked 研究文件（使用者裁定 commit 進 docs/）。

## 2. 範圍

**In scope**

- 移除 kit 內所有 gemini 活引用（agent、script、hook 廣播、settings、
  init.sh 檢查、README、ARCHITECTURE、kit-workflow、templates）。
- 新 `research-scout` agent（Claude 原生）與 `research-before-planning`
  的呼叫對象替換。
- Profile 語義簡化：profile 從此只決定 reviewer。
- fix-later 三項修復。
- smoke.sh / hooks-smoke.sh 同步與新增斷言。
- `--update` 遷移提示（gemini 孤兒檔）。
- VERSION → 3.4.0，文件版本段同步。

**Out of scope**

- `docs/superpowers/` 歷史 plan/spec（紀錄文件不回改）。
- P2 大項（kit plugin 化、codex plugin 自動安裝、mise 深化）。
- `git push origin main`（需使用者憑證，完工後提醒）。

## 3. 設計

### 3.1 research-scout（換心臟不換介面）

新檔 `.claude/agents/research-scout.md`，**刪除** `gemini-research-scout.md`：

- frontmatter：`name: research-scout`、`tools: WebSearch, WebFetch, Read`、
  `model: sonnet`（使用者裁定：研究品質/成本平衡點；舊檔 haiku 的前提
  「只轉接 CLI」已不成立）。
- description 沿用觸發語義（brainstorming/planning 前、不熟 library /
  security / perf / 新穎架構；只研究，不寫 code、不 review），**移除
  「Full profile only」**。
- 工作流：父代理供 Topic / Context / 2-5 個具體問題 / Constraints →
  scout 以 WebSearch 多角度查詢 + WebFetch 細讀權威來源（官方文件、
  changelog、postmortem）→ 綜合輸出。
- 輸出格式沿用：Recommended approach / Key findings / Pitfalls /
  References / My note to parent（confidence、不確定處、建議追查）。
- 新增研究紀律：優先官方來源；發現註明資訊日期；區分「已 fetch 原文
  驗證」與「僅搜尋摘要」。
- 失敗處理：只剩「WebSearch 不可用 → 回報父代理決定」；研究照舊
  nice-to-have、不是 gate。

`research-before-planning/SKILL.md`：spawn 對象換成 `research-scout`，
移除 full-profile-only 註記與 GEMINI_API_KEY 失敗分支，其餘工作流
（先寫問題→spawn→整合→衝突即停→交棒 superpowers）逐字保留。

### 3.2 Profile 語義簡化

| profile | 研究 | 審查 |
|---------|------|------|
| `full` | research-scout（Claude 原生） | Codex（跨模型隔離） |
| `solo` | 同上（不再 full 限定） | fresh-context 自審（狀態/時間隔離） |

- `session-start.sh` KIT_CONTEXT：full 只報 `reviewer=codex(...)`；
  researcher 一項移除（已內建，無外部可用性可言）。solo 訊息維持。
- `kit-workflow.md`：`research (full: gemini scout)` → `research
  (research-scout subagent)`；STOP 條件 `(full) codex/gemini unavailable`
  → 只提 codex。

### 3.4 fix-later 三項

1. **`--help` 噪音**：`grep '^#' "$0"` 印出全腳本註解。改為只印檔頭
   區塊（第 2 行起、遇第一個非 `#` 行即停）。
2. **settings.json deploy-if-absent**：`--update` 增加「目標無
   `.claude/settings.json` → 補上模板」；已存在則維持現行 diff 提示、
   絕不覆蓋。
3. **smoke env 過濾**：`init_run` 以 `env -u KIT_PROFILE` 隔離宿主
   環境變數，讓 environment check 輸出確定化（GEMINI_API_KEY 隨
   gemini 移除已無關）。

### 3.5 遷移路徑（已鋪 kit 的專案）

`--update` 比照 PROMPTING.md 前例，偵測到目標有
`gemini-research-scout.md` 或 `gemini_exec.sh` 時印針對性提示：
kit 已不再整合 gemini，這兩檔可刪，settings.json 的 gemini_exec
allow 行可自行移除。孤兒偵測本來就會列出這兩檔，此提示只是講明
「為什麼變孤兒」。不代刪任何檔。
