# Kit v4.5 — proj html dashboard 設計

日期：2026-07-11
狀態：已與 user 確認（開啟方式=自動開瀏覽器；視覺=極簡無彩+語意色；花錢區不強行加總）

## 1. 問題

`proj`（v4.4）的終端表格在寬欄位（`status_note` 動輒 50+ 中文字、付費服務的 `cancel` 更長）下會被終端硬 wrap，多欄一起 wrap 就糊成一團，看不清全貌。表格對齊邏輯正確，是內容量對 80 字元終端這個載體太大。user 要「視覺化」。

## 2. 解法

新增 `proj html` 子指令：掃描各專案 PROJECT.toml，產出**自包含單檔 HTML** dashboard，寫到 `~/.local/share/proj/dashboard.html`，在 WSL 下自動用 `explorer.exe` 開 Windows 預設瀏覽器。

**範圍外**：常駐 server、即時金額 billing API、非 proj 資料源。dashboard 是每次執行重新掃描的靜態快照。

## 3. 指令行為

- `proj html`：掃描 → 產檔 → 開啟。
- 產檔路徑固定 `~/.local/share/proj/dashboard.html`（目錄不存在則 `mkdir -p`）。
- 開啟：`shutil.which("explorer.exe")` 偵測 WSL；有則 `subprocess.run(["explorer.exe", win_path])`，`win_path` 由 `wslpath -w <path>` 取得。偵測不到（非 WSL）→ 僅 stderr 印檔案路徑，exit 0，不報錯。
- explorer.exe 對 `wslpath -w` 得到的 UNC/磁碟路徑開啟；轉換失敗則退回印路徑。

## 4. 版面（三區，對應 user 三大痛點）

1. **頂部 KPI 帶**：專案總數、各狀態計數（active/mvp/done/paused/abandoned）、固定月費項目數、按用量待盯數。stat-tile 風格。
2. **專案卡片網格**：每張卡 = 名稱、狀態徽章（語意色）、`status_note`、起始指令（`[commands]` 逐條，一鍵複製）、付費服務小標、最後 commit 日期、過時 ⚠。頂部狀態徽章可點擊篩選卡片（inline JS）。
3. **花錢區**：三組——
   - **固定月費**：`billing` 為「月費」或「年費」的項目聚在一起。**不強行加總**（`monthly_est` 是自由文字、幣別/週期混雜，加總會出錯）；改為逐項列出讓 user 一眼看到（rss.app NT$260、網域 NT$260/年）。
   - **按用量待盯**：`billing` 為「按用量」的項目，標紅提醒（OpenAI/Gemini/GCS）。
   - **free-tier $0**：可摺疊（`<details>`），佔位小。

## 5. 視覺

照 user 既有極簡無彩審美（blog/portfolio 風格）：

- 中性灰階為底，狀態才用克制的語意色：active=綠、mvp=藍、done=灰、paused=琥珀、abandoned=更淡灰。
- 明暗主題都支援（`prefers-color-scheme`）。
- 卡片 grid 響應式（`repeat(auto-fill, minmax(...))`），窄螢幕單欄。
- 色彩依 dataviz 指引（palette validator）挑選，KPI tiles 視覺一致。
- 字體用系統字（無外部字型請求，維持自包含）。

## 6. 互動（inline JS，維持自包含）

- 狀態篩選：點 KPI 帶的狀態徽章 → 只顯示該狀態卡片，再點取消。
- 起始指令複製：卡片內每條指令旁一個複製鈕（`navigator.clipboard.writeText`）。
- 無 JS 也能讀（漸進增強）：篩選/複製是加值，靜態內容全可見。

## 7. 技術

- HTML 生成：純 Python 字串組裝（模板函式 + `html.escape`），不引第三方模板引擎（維持零依賴）。
- 所有使用者資料經 `html.escape` 轉義（status_note/指令/服務名可能含 `<`、`&`）。
- CSS/JS 全 inline 在 `<style>` / `<script>`，無外部 `src`/`href` 到 http(s)。
- 資料來源：即時掃描結果嵌入 HTML（自包含快照）。

## 8. 錯誤處理

- 壞 TOML：卡片顯示紅色「manifest 損壞」，不炸整頁（沿用 cmd_list 的 (data, err) 分支）。
- 未登記專案：灰卡，只有名稱 + 最後 commit。
- 非 WSL / 無 explorer.exe：產檔成功，stderr 印路徑，exit 0。
- `wslpath` 缺失或轉換失敗：印路徑，不報錯。

## 9. 測試（tests/proj-smoke.sh 增補）

- `proj html` 產出 `$HOME/.local/share/proj/dashboard.html` 存在（測試用 `HOME` 指向 fixture）。
- 內容含關鍵字：專案名（good-proj）、狀態值（mvp）、月費字樣、付費服務名。
- **自包含**：`grep` 確認產檔無 `http://`、`https://`、`src=`、`<link`（外部引用零）。
- 壞 TOML 專案在頁面顯示「manifest 損壞」而非缺席或炸掉。
- 非 WSL 降級：stub PATH（無 explorer.exe）下 exit 0 且產檔。
- HTML 轉義：含 `<`/`&` 的 status_note 不破版（fixture 加一個含特殊字元的專案，斷言原始 `<script>` 不出現在輸出、`&lt;` 出現）。

## 10. 版本與文件

- `VERSION`：4.4.0 → 4.5.0。
- kit `README.md`「專案總覽:proj」節加一行 `proj html`（產出視覺化 dashboard，自動開瀏覽器）。
- `bin/proj` 的 `__doc__` 用法區加 `proj html`。

## 11. 部署

proj 是 `~/.local/bin/proj` → kit `bin/proj` 的 symlink，改 `bin/proj` 立即生效。11 個既有專案**不需要**任何動作（HTML 生成是 proj 內建功能，不涉及各專案檔案）。

## 12. 驗收標準

- `tests/proj-smoke.sh` 與 `tests/smoke.sh` 全綠。
- `proj html` 在 WSL 下產檔並自動開瀏覽器，一頁看到 12 專案狀態、起始指令、花錢全貌，不受終端寬度限制。
- 視覺符合極簡審美、明暗都可讀、窄螢幕不破版。
